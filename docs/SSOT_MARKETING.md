## 3. USP（Unique Selling Proposition）

### 3-1. メインUSP ✅ 確定

```
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  「現場の"見えない"を、"見える"に。」                          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**コンセプト**:
現場トラブルの大半は「見えない」ことが原因。
- 人の予定が見えない
- 在庫の有無・過不足が見えない
- 車両関連の法務遵守状況が見えない
- 図面の変更点、関係各所への認識が見えない

これらが「見える」ようになることで、トラブルを未然に防ぐ。

### 3-2. サブUSP（製品別）✅ 確定

| 製品 | サブUSP | 見えるようになるもの |
|------|---------|-------------------|
| **ミエルボード** | 予定がミエル | 誰がどこで何をしているか |
| **ミエルストック** | 在庫がミエル | 資材の有無・過不足 |
| **ミエルドライブ** | 車両がミエル | 配車・アルコールチェック状況 |
| **ミエルファイル** | 変更と履歴がミエル | 図面・資料の変更検知・施工履歴 |

### 3-3. タグライン

```
「見えないから起こるトラブル、ミエルで終わりにする。」
```

### 3-4. ブランドプロミス（3つの約束）

| 約束 | 内容 |
|------|------|
| **1. 見える** | 必要な情報がいつでもどこでも見える |
| **2. 簡単** | 導入初日から全員が使える |
| **3. 安心** | 今の運用を壊さず、進化させる |

## 8. グロース戦略 ✅ 確定

### 8-1. PQL（Product Qualified Lead）定義

無料トライアルからの有料転換を最大化するため、**購入意欲が高いユーザー行動**を定義する。

```
┌─────────────────────────────────────────────────────────────┐
│  PQL（Product Qualified Lead）定義                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  【有料転換が見込める行動】※ 2つ以上でPQL認定               │
│  ✓ 3人以上のユーザーを追加                                   │
│  ✓ 1週間以上のスケジュールを登録                             │
│  ✓ サイネージ表示を1回以上利用                               │
│  ✓ 3日以上連続ログイン                                       │
│                                                              │
│  【PQL認定時のアクション】                                   │
│  ・自動メール: 「〇〇さん、ご利用ありがとうございます」      │
│  ・CS介入: 電話/デモ提案（Day 7〜10）                        │
│  ・インセンティブ: 「今週中の契約で初月50%OFF」              │
│                                                              │
│  【非PQL（転換見込み低）への対応】                           │
│  ・Day 7: リマインドメール「使い方で困っていませんか？」     │
│  ・Day 10: 電話サポート提案                                  │
│  ・Day 13: 最終リマインド「明日でトライアル終了です」        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 8-8. スキーマ方針（グロース機能）

**原則: metadata拡張で対応し、スキーマ変更は最小限に留める**

| 機能 | 実装方式 | 備考 |
|------|---------|------|
| 行動トラッキング | Organization.metadata / User.metadata 拡張 | 集計値のみ保存 |
| PQL判定 | Organization.metadata 拡張 | pqlStatus, pqlQualifiedAt |
| ヘルススコア | Organization.metadata 拡張 | healthScore, healthScoreHistory |
| NPS | Organization.metadata 拡張 | npsScore, npsCategory |
| 紹介 | Organization.metadata 拡張 | referralCode |
| 詳細ログ | 外部サービス（Mixpanel等） | プロダクト本体には保存しない |

**スキーマ変更が必要になった場合**:
- Year 2 Q1時点で再評価
- 必要最小限のテーブル追加を検討
- 管理AI承認を必須とする
