## 7. 設計AI・管理AIへの指示

### 7-1. 設計判断の優先順位

すべての設計判断において、以下の優先順位を適用してください：

```
1️⃣ M&A価値への貢献
   └─ ANDPADが欲しがる機能か？freeeに刺さるか？

2️⃣ ターゲット顧客のペイン解決
   └─ 「見えない」問題を解決できるか？

3️⃣ ARR・顧客数への貢献
   └─ 新規獲得？アップセル？解約防止？

4️⃣ 実装コスト・リスク
   └─ 既存基盤を活かせるか？技術的リスクは？
```

### 7-2. 機能設計時のチェックリスト

新しい機能を設計・実装する際は、以下を確認してください：

- [ ] ANDPADにとっての価値は何か？
- [ ] freeeにとっての価値は何か？
- [ ] 「見えない → 見える」を実現しているか？
- [ ] ターゲット顧客（弱電・電気工事 10〜50名）に刺さるか？
- [ ] 競合（ANDPAD/SPIDERPLUS）との差別化になっているか？
- [ ] アップセル・クロスセルの機会を生み出せるか？

### 7-3. 避けるべきこと

- ❌ ANDPADと同じ機能を作る（図面閲覧、ピン・マーキング等）
- ❌ ターゲット外の業種向け機能（介護、物流等は Phase 3以降）
- ❌ エンタープライズ向けの複雑なカスタマイズ機能
- ❌ 既存SSOTの料金体系・機能スコープとの矛盾

## 📋 このプロンプトの使い方

### 設計AIの場合

1. 新しい設計AIチャットを開始
2. **このプロンプト**（`AI_ROLES/STRATEGIC_CONTEXT_PROMPT.md`）を読み込み
3. 対象のSSOT（例: `SSOT_GENBA_WEEK.md`）を読み込み
4. `.cursorrules` / `docs/TAG_COMMANDS.md`（必要に応じて）を読み込み
5. `>> write` または `>> impl` タグで作業開始

### 管理AIの場合

1. 新しい管理AIチャットを開始
2. **このプロンプト**（`AI_ROLES/STRATEGIC_CONTEXT_PROMPT.md`）を読み込み
3. `docs/QUALITY_MANAGEMENT_OVERVIEW.md` / `docs/DONE_DEFINITION.md`（必要に応じて）を読み込み
4. `>> next`（※Plane API連携が未実装の場合は手動運用）または PR レビュー開始
