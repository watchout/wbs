## 1. 指標体系

### 1.1 North Star Metric（最重要指標）

> プロダクトの成功を1つの数字で表す指標

| 項目 | 内容 |
|------|------|
| **指標名** | |
| **定義** | |
| **計算式** | |
| **目標** | |

**例**:
- Slack: 「週あたりのメッセージ送信数」
- Airbnb: 「予約泊数」
- Spotify: 「月間リスニング時間」

## 4. 収益指標

### 4.1 基本収益指標

| 指標 | 定義 | 計算式 | 目標 | 頻度 |
|------|------|-------|------|------|
| MRR | 月間経常収益 | Σ(月額課金) | | 月次 |
| ARR | 年間経常収益 | MRR × 12 | | 月次 |
| 有料顧客数 | 課金中の顧客数 | | | 月次 |
| ARPU | 1顧客あたり月間収益 | MRR ÷ 有料顧客数 | | 月次 |

### 4.2 成長指標

| 指標 | 定義 | 計算式 | 目標 | 頻度 |
|------|------|-------|------|------|
| Net New MRR | 純増MRR | 新規 + 拡大 - 縮小 - 解約 | | 月次 |
| New MRR | 新規顧客からのMRR | | | 月次 |
| Expansion MRR | 既存顧客の拡大MRR | | | 月次 |
| Contraction MRR | 既存顧客の縮小MRR | | | 月次 |
| Churned MRR | 解約MRR | | | 月次 |
| MRR成長率 | 月間成長率 | (今月MRR - 先月MRR) ÷ 先月MRR | | 月次 |

### 4.3 MRRの内訳

```
MRR Movement

前月MRR:     ¥____________

+ New MRR:         ¥____________  (新規)
+ Expansion MRR:   ¥____________  (アップグレード)
- Contraction MRR: ¥____________  (ダウングレード)
- Churned MRR:     ¥____________  (解約)
─────────────────────────────────
= Net New MRR:     ¥____________

今月MRR:     ¥____________
```

## 5. 顧客価値指標

### 5.1 LTV関連

| 指標 | 定義 | 計算式 | 目標 | 頻度 |
|------|------|-------|------|------|
| LTV | 顧客生涯価値 | ARPU × 平均継続月数 | | 月次 |
| 平均継続期間 | 解約までの平均月数 | 1 ÷ 月次解約率 | | 月次 |
| LTV/CAC | LTVとCACの比率 | LTV ÷ CAC | 3.0以上 | 月次 |
| CAC回収期間 | CACを回収するまでの月数 | CAC ÷ ARPU | 12ヶ月以内 | 月次 |

### 5.2 解約関連

| 指標 | 定義 | 計算式 | 目標 | 頻度 |
|------|------|-------|------|------|
| 月次解約率 | 月間の解約率 | 解約顧客数 ÷ 月初顧客数 | 5%以下 | 月次 |
| 収益解約率 | MRRベースの解約率 | 解約MRR ÷ 月初MRR | 3%以下 | 月次 |
| Net Revenue Retention | 既存顧客の収益維持率 | (拡大 - 縮小 - 解約) ÷ 前月MRR | 100%以上 | 月次 |

## 6. 紹介指標

### 6.1 バイラル指標

| 指標 | 定義 | 計算式 | 目標 | 頻度 |
|------|------|-------|------|------|
| 紹介率 | 紹介を実施した顧客の割合 | 紹介実施者 ÷ 全顧客 | 10%+ | 月次 |
| 平均紹介数 | 1人あたりの紹介人数 | 紹介総数 ÷ 紹介実施者 | 3+ | 月次 |
| 紹介CV率 | 紹介からの登録転換率 | 紹介経由登録 ÷ 紹介クリック | 30%+ | 月次 |
| バイラル係数 | バイラル成長係数 | 平均紹介数 × 紹介CV率 | 0.5+ | 月次 |
| NPS | 推奨度 | 推奨者% - 批判者% | 50+ | 月次 |

## 8. アラート設定

### 8.1 アラート基準

| 指標 | 警告（黄） | 危険（赤） | 対応 |
|------|-----------|-----------|------|
| 日次登録数 | 目標の80%未満 | 目標の50%未満 | 流入チャネル確認 |
| CV率 | 3%未満 | 2%未満 | LP改善 |
| アクティベーション率 | 30%未満 | 20%未満 | オンボーディング改善 |
| 月次解約率 | 7%超 | 10%超 | 解約理由分析 |

### 8.2 通知設定

| 条件 | 通知先 | 方法 |
|------|-------|------|
| 危険アラート | 全員 | Slack + メール |
| 警告アラート | 担当者 | Slack |
| 週次サマリー | 全員 | メール |

## 9. レビュープロセス

### 9.1 日次レビュー（5分）

- [ ] 日次指標の確認
- [ ] 異常値の有無確認
- [ ] 必要に応じて対応

### 9.2 週次レビュー（30分）

- [ ] 週次指標の確認
- [ ] 目標との差分分析
- [ ] 改善アクションの決定

### 9.3 月次レビュー（2時間）

- [ ] 月次指標の確認
- [ ] コホート分析
- [ ] LTV/CAC分析
- [ ] 次月目標の設定
- [ ] 戦略の見直し

## 10. ツール・計測方法

### 10.1 計測ツール

| 目的 | ツール | 対象指標 |

| Web分析 | Google Analytics | 訪問者、CV |

| イベント名 | トリガー | プロパティ |

| subscription_started | 課金開始 | plan, price |
| subscription_cancelled | 解約 | reason |
