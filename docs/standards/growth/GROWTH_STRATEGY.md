## 3. Activation（活性化）戦略

### 3.1 アクティベーションの定義

> 「ユーザーが価値を感じた」と判断できる行動

| レベル | 行動 | 目標到達率 |
|-------|------|-----------|
| Level 1 | 登録完了 | 100%（基準） |
| Level 2 | 初期設定完了 | % |
| Level 3 | 最初の[主要アクション]完了 | % |
| **Activated** | [価値を感じる行動]完了 | % |

### 3.2 オンボーディング設計

```
Day 0: 登録 → ウェルカム → 初期設定
          ↓
Day 1: 最初のタスク完了ガイド
          ↓
Day 2-3: 主要機能の案内
          ↓
Day 7: 成果確認 + 次のステップ
```

| ステップ | 目標 | 施策 |
|---------|------|------|
| 初期設定 | 5分以内に完了 | プログレスバー、スキップ可能 |
| 最初のタスク | 成功体験を得る | チュートリアル、テンプレート |
| 主要機能 | 価値を理解 | ツールチップ、動画 |

### 3.3 アクティベーション改善

| 離脱ポイント | 原因仮説 | 改善策 |
|-------------|---------|-------|
| 登録後すぐ | 価値がわからない | ウェルカム動画 |
| 初期設定中 | 面倒 | ステップ削減 |
| 最初のタスク | 難しい | テンプレート提供 |

### 6.2 バイラル係数

```
バイラル係数 = 平均紹介数 × 紹介からのCV率

例:
- 1人のユーザーが平均3人に紹介
- 紹介された人の20%が登録

バイラル係数 = 3 × 0.2 = 0.6

※ 1.0以上で自己増殖（指数関数的成長）
```
