## 2. LP構成（PASONAフレームワーク）

### 全体構成

```
┌─────────────────────────────────────────────┐
│ 1. ファーストビュー（Problem）              │  ← 3秒で離脱を防ぐ
│    ヘッドライン + サブヘッド + CTA           │
├─────────────────────────────────────────────┤
│ 2. 問題の深掘り（Agitation）                │  ← 共感を得る
│    痛みの具体化                              │
├─────────────────────────────────────────────┤
│ 3. 解決策の提示（Solution）                 │  ← 希望を見せる
│    製品紹介 + 主要機能                       │
├─────────────────────────────────────────────┤
│ 4. ベネフィット + 社会的証明（Offer）       │  ← 信頼を構築
│    得られる結果 + 証言 + 実績                │
├─────────────────────────────────────────────┤
│ 5. 限定性・緊急性（Narrow down）            │  ← 今動く理由
│    限定特典 + 期限                           │
├─────────────────────────────────────────────┤
│ 6. CTA + FAQ（Action）                      │  ← 行動を促す
│    登録フォーム + よくある質問               │
└─────────────────────────────────────────────┘
```

### 3.6 CTA + FAQ（Action）

**目的**: 最後の一押し、不安を解消

#### メインCTA

| 要素 | 内容 |
|------|------|
| ボタンテキスト | 「今すぐ無料で始める」 |
| ボタン色 | コントラストの高い色 |
| マイクロコピー | 「登録は30秒」「クレカ不要」 |
| 配置 | ページ内に3-5回 |

#### 登録フォーム

| フィールド | 必須 | 理由 |
|-----------|------|------|
| メールアドレス | ✅ | 連絡手段 |
| 名前 | ☐ | パーソナライズ |
| 会社名 | ☐ | B2Bの場合 |
| 電話番号 | ☐ | 営業する場合のみ |

**原則**: フィールドは最小限に（多いほどCV率低下）

## 6. A/Bテスト計画

### 6.1 テスト優先順位

| 優先度 | テスト要素 | 仮説 |
|-------|-----------|------|
| 高 | ヘッドライン | 痛み訴求 vs 結果訴求 |
| 高 | CTAボタンテキスト | 「無料で始める」vs「今すぐ登録」 |
| 中 | ファーストビュー画像 | 製品画面 vs 人物 |
| 中 | 社会的証明の配置 | 上部 vs 下部 |
| 低 | ボタン色 | オレンジ vs 緑 |

### 6.2 テスト実施基準

| 指標 | 最小サンプル数 | 有意水準 |
|------|--------------|---------|
| CV率 | 各パターン100件以上 | 95% |
