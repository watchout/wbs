## 1. ローンチ戦略概要

### 1.1 プロダクトローンチ・フォーミュラ

```
  Pre-Pre        Pre           Launch        Post
   Launch       Launch                      Launch
 ─────────────────────────────────────────────────→
    2週間前      1週間前        3-7日        1週間後

    種まき    →  期待醸成  →  販売期間  →  フォロー
    好奇心       欲求喚起     緊急性        継続価値
```

### 1.2 各フェーズの目的

| フェーズ | 目的 | 主な施策 |
|---------|------|---------|
| Pre-Pre Launch | 好奇心を喚起 | ティーザー、アンケート |
| Pre Launch | 欲求を最大化 | 価値コンテンツ、社会的証明 |
| Launch | 今すぐ行動を促す | 限定オファー、緊急性 |
| Post Launch | 成功を支援 | オンボーディング、サポート |

## 3. Pre-Pre Launch（2週間前〜）

### 3.1 目的

- 好奇心を喚起する
- 「何かが起きる」という期待感を作る
- 見込み客のリストを温める

### 3.2 施策一覧

| 日付 | 施策 | チャネル | 担当 | ステータス |
|------|------|---------|------|-----------|
| -14日 | ティーザー投稿「何か新しいことを始めます」 | Twitter | | ☐ |
| -14日 | ティーザーメール送信 | メール | | ☐ |
| -12日 | アンケート「どんな機能が欲しい？」 | Twitter | | ☐ |
| -10日 | カウントダウン開始 | Twitter/LP | | ☐ |
| -10日 | 開発の裏側を公開 | Twitter | | ☐ |

### 3.3 ティーザーコンテンツ

**SNS投稿テンプレート**:
```
【予告】

2週間後、
あなたの[課題]を解決する
新しい[カテゴリ]を公開します。

詳細は追ってお知らせします。

楽しみにしていてください👀

#[ハッシュタグ]
```

**メールテンプレート**:
```
件名: 【予告】新しいお知らせがあります

[名前]さん

2週間後、
あなたに特別なお知らせがあります。

今はまだ詳しくお伝えできませんが、
[課題]で悩んでいる方には
きっと喜んでいただけるものです。

楽しみにしていてください。

[署名]
```

## 4. Pre Launch（1週間前〜）

### 4.1 目的

- 欲求を最大化する
- 価値を事前に証明する
- 社会的証明を提示する

### 4.2 施策一覧

| 日付 | 施策 | チャネル | 担当 | ステータス |
|------|------|---------|------|-----------|
| -7日 | 価値コンテンツ①配信 | メール/SNS | | ☐ |
| -5日 | 価値コンテンツ②配信 | メール/SNS | | ☐ |
| -3日 | 社会的証明（β版ユーザーの声） | SNS | | ☐ |
| -3日 | ローンチ詳細メール | メール | | ☐ |
| -1日 | 最終予告「明日公開」 | 全チャネル | | ☐ |
| -1日 | ウェビナー/ライブ（任意） | YouTube/Twitter | | ☐ |

### 4.3 価値コンテンツ

**コンテンツ①: 問題の深掘り**
```
件名: ○○が上手くいかない本当の理由

[名前]さん

[課題]で悩んでいませんか？

実は、その原因は
多くの人が見落としている
[根本原因]にあります。

[原因の説明]
[解決の方向性]

次回は、具体的な解決策を
お伝えします。

[署名]
```

**コンテンツ②: 解決策の提示**
```
件名: [課題]を解決する方法

[名前]さん

前回、[課題]の原因について
お話ししました。

今日は、その解決策を
具体的にお伝えします。

[解決策の説明]
[製品がどう役立つか]

いよいよ明後日、
[製品名]を公開します。

詳細は明日お伝えします。

[署名]
```

## 5. Launch（ローンチ当日〜7日間）

### 5.1 目的

- 今すぐ行動を促す
- 限定性・緊急性で決断を後押し
- 購入者をすぐにサポート

### 5.2 施策一覧

| 日付 | 施策 | チャネル | 担当 | ステータス |
|------|------|---------|------|-----------|
| Day 0 | ローンチ告知（朝） | 全チャネル | | ☐ |
| Day 0 | ローンチメール①（告知） | メール | | ☐ |
| Day 1 | ローンチメール②（詳細） | メール | | ☐ |
| Day 1 | 登録者の声をシェア | SNS | | ☐ |
| Day 3 | ローンチメール③（FAQ） | メール | | ☐ |
| Day 5 | 締め切り予告 | 全チャネル | | ☐ |
| Day 7 | 最終メール（締め切り） | メール | | ☐ |
| Day 7 | 締め切り投稿（当日） | SNS | | ☐ |

### 5.3 ローンチメール

**Day 0: ローンチ告知**
```

[限定特典の説明]

[CTAボタン]

**Day 7: 締め切り**
```

・価格が[現在]→[通常]に

### 5.4 限定オファー設計

| 要素 | 内容 |
|------|------|
| **早期割引** | 通常価格の○%OFF |
| **限定特典** | [特典1]、[特典2] |
| **数量限定** | 先着○名様限定 |
| **期間限定** | ○月○日23:59まで |
| **保証** | ○日間返金保証 |

## 6. Post Launch（ローンチ後）

### 6.1 目的

- 新規ユーザーの成功を支援
- 成功事例を収集
- 次のローンチへの布石

### 6.2 施策一覧

| 日付 | 施策 | 担当 | ステータス |
|------|------|------|-----------|
| +1日 | ウェルカムメール送信 | | ☐ |
| +1日 | オンボーディング開始 | | ☐ |
| +3日 | フォローアップメール（使い方） | | ☐ |
| +7日 | フィードバック依頼 | | ☐ |
| +14日 | 成功事例インタビュー依頼 | | ☐ |
| +30日 | 証言・レビュー依頼 | | ☐ |

### 6.3 オンボーディング

| ステップ | 内容 | 目標 |
|---------|------|------|
| Step 1 | アカウント設定 | 5分以内に完了 |
| Step 2 | 最初のタスク完了 | 「できた！」を体験 |
| Step 3 | 主要機能を体験 | 価値を実感 |
| Step 4 | 習慣化 | 日常に組み込む |
