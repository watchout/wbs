# 06_FULL_LIFECYCLE.md - プロダクトライフサイクル全体定義

> アイデア→検証→開発→リリース→グロースの全フェーズを定義

---

## フレームワークの目的（再定義）

```
「ぼんやりとしたアイデア」を
  ↓
「AIが実装可能な明確な仕様」に変換し
  ↓
「開発中からリードを獲得」しながら
  ↓
「最短でサービスとしてリリース」し
  ↓
「売れる状態」でスタートする
```

---

## フェーズ別の詳細

### Phase -1: アイデア明確化（1-2週間）

**目的**: ぼんやりしたアイデアを言語化・構造化する

| 成果物 | 説明 | 重要度 |
|-------|------|-------|
| IDEA_CANVAS.md | アイデアキャンバス | 必須 |
| COMPETITOR_ANALYSIS.md | 競合分析 | 必須 |
| USER_PERSONA.md | ユーザーペルソナ | 必須 |

**完了条件**:
- [ ] 解決する課題が1文で説明できる
- [ ] ターゲットユーザーが具体的に定義されている
- [ ] 競合との差別化ポイントが明確
- [ ] 「なぜ今これを作るか」が説明できる

---

### Phase 0: 要件定義（1-2週間）

**目的**: アイデアを実装可能な要件に変換

| 成果物 | 説明 | 重要度 |
|-------|------|-------|
| SSOT-0_PRD.md | プロダクト要件定義書 | 必須 |
| SSOT-1_FEATURE_CATALOG.md | 機能カタログ | 必須 |
| VALUE_PROPOSITION.md | 価値提案キャンバス | 必須 |

**完了条件**:
- [ ] MVP機能が特定されている
- [ ] 価値提案が明確
- [ ] P0機能が3-5個に絞られている

---

### Phase 0.5: 市場検証・リード獲得開始（1-2週間）

**目的**: 開発前に市場の反応を確認し、リード獲得を開始

| 成果物 | 説明 | 重要度 |
|-------|------|-------|
| LANDING_PAGE_SPEC.md | LP仕様書 | 必須 |
| LP実装 | 実際のLP | 必須 |
| BETA_PROGRAM.md | βプログラム設計 | 必須 |
| SNS_STRATEGY.md | SNS運用戦略 | 必須 |

**完了条件**:
- [ ] LPが公開されている
- [ ] β版申し込みフォームが動作している
- [ ] SNSアカウントが稼働開始
- [ ] 最初のリードが獲得できている（目標: XX件）

**検証ポイント**:
- LP訪問者のコンバージョン率
- β申し込み数の推移
- SNSでの反応

---

### Phase 1-5: 開発（既存フレームワーク）

**並行タスク**:
- 週次の開発進捗発信
- β申し込み者へのアップデート
- SNSでの継続的な発信

## 「作っても売れない」を防ぐ仕組み

### 1. Phase -1 で検証

```
✅ そもそも課題は存在するか？
✅ その課題を持つ人は十分にいるか？
✅ 既存の解決策で満足していないか？
✅ お金を払う価値があるか？
```

### 2. Phase 0.5 で検証

```
✅ LPを見て興味を持つか？（訪問者数）
✅ β版に申し込むか？（CV率）
✅ どんな人が申し込むか？（ペルソナ検証）
✅ 申し込み理由は何か？（価値提案検証）
```

### 3. 開発中も継続検証

```
✅ SNSでの反応は？
✅ β申し込み数は増えているか？
✅ 待ちリストの人は本当に待っているか？
```

## 開発中からリードを取る仕組み

### ステップ1: LP公開（開発開始と同時）

```
LP内容:
- 課題の提示（共感）
- ソリューションの概要
- 主要機能（スクリーンショット/モック）
- 「β版に申し込む」CTA
- 開発ロードマップ
```

### ステップ2: SNS運用

```
発信内容:
- 開発の進捗（#buildinpublic）
- 機能のチラ見せ
- 課題・学びの共有
- ユーザーの声（β申し込み者）
```

### ステップ3: β申し込み者への継続コミュニケーション

```
メール/Discord:
- 週次の開発アップデート
- 優先アクセスの案内
- フィードバック募集
- 早期割引の案内
```

## 判断ポイント

### Phase 0.5 → Phase 1 への移行判断

```
Go条件:
✅ β申し込みが目標の50%以上
✅ CV率が5%以上
✅ 申し込み理由が価値提案と一致

No-Go条件（ピボット検討）:
❌ β申し込みが目標の20%未満
❌ CV率が2%未満
❌ 申し込み理由が想定と大きく異なる
```
