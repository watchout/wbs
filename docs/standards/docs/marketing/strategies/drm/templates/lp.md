## LP構成（上から順）

### 1. ファーストビュー

```markdown
## {{キャッチコピー}}

{{サブコピー: ベネフィットを1文で}}

[{{CTA}}]({{URL}})

<!-- 参照: VALUE_PROPOSITION.md のメインメッセージ -->
```

### 2. 問題提起

```markdown

- {{ペインポイント1}}
- {{ペインポイント2}}
- {{ペインポイント3}}

<!-- 参照: users/pain-points.md -->
```

### 3. 解決策の提示

```markdown
## {{プロダクト名}}が解決します

2. **{{特徴2}}**: {{説明}}
3. **{{特徴3}}**: {{説明}}

<!-- 参照: SSOT-0_PRD.md §3, §5 -->
```


## AI生成プロンプト

```
以下の情報を元に、LP用のコピーを生成してください。

【参照ドキュメント】
- VALUE_PROPOSITION.md: メインメッセージ
- SSOT-0_PRD.md §3: 価値提案
- SSOT-0_PRD.md §5: 機能一覧
- users/pain-points.md: ペインポイント
- users/personas.md: ターゲット詳細
- market/competitors.md: 差別化ポイント

【出力形式】
1. キャッチコピー（10字以内）
2. サブコピー（30字以内）
3. 問題提起（3点）
4. 特徴（3点、各50字以内）
5. CTA文言（5字以内）

【トーン】
- {{ターゲットに合わせたトーン指定}}
```

---

## レビューチェックリスト

- [ ] ファーストビューで価値が伝わる
- [ ] CTAが目立つ、1種類のみ
- [ ] ペインポイントがターゲットに刺さる
- [ ] 特徴がベネフィットとして表現されている
- [ ] リスクリバーサルがある
- [ ] 緊急性/希少性の要素がある
- [ ] モバイルで読みやすい

---

## 成功パターン

### パターンA: 問題解決型

```
問題 → 共感 → 解決策 → 証拠 → CTA
```

### パターンB: ベネフィット先行型

```
ベネフィット → 理由 → 証拠 → リスクリバーサル → CTA
```

### パターンC: ストーリー型

```
課題を抱えた主人公 → 解決策との出会い → 変化 → あなたも → CTA
```
