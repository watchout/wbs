# 問題提起記事テンプレート

> PASONA の P（Problem）を深掘りする記事構成

---

## 記事構成

### 1. タイトル

```
【{{ターゲット}}必見】{{問題}}を放置すると{{悪い結果}}になる理由
```

### 2. リード文

```markdown
{{ターゲット}}の皆さん、{{問題}}で悩んでいませんか？

実は、この問題を放置すると{{悪い結果}}につながる可能性があります。

この記事では、{{問題}}の本質と、なぜ今すぐ対処すべきかを解説します。
```

### 3. 問題の具体化

```markdown
## {{問題}}とは何か

{{問題の定義と背景}}

### よくある症状

- {{症状1}}
- {{症状2}}
- {{症状3}}

<!-- 参照: users/pain-points.md -->
```

### 4. 問題の深刻さ

```markdown
## なぜ{{問題}}は深刻なのか

### 1. {{理由1}}

{{説明}}

### 2. {{理由2}}

{{説明}}

### 3. {{理由3}}

{{説明}}
```

### 5. 放置した場合の結果

```markdown
## {{問題}}を放置するとどうなるか

### 短期的な影響

{{短期的な悪影響}}

### 長期的な影響

{{長期的な悪影響}}
```

### 6. 解決への希望

```markdown
## でも、解決方法はあります

{{問題}}は解決可能です。

次回の記事で、具体的な解決策をご紹介します。

[解決策の記事を読む]({{次の記事URL}})
```

---

## AI生成プロンプト

```
以下の情報を元に、問題提起記事を生成してください。

【参照ドキュメント】
- users/pain-points.md: 具体的な問題
- users/personas.md: ターゲット詳細
- SSOT-0_PRD.md §3: 解決する課題

【制約】
- 文字数: 2000-3000字
- 見出し: H2を4-6個
- 専門用語には解説を付ける

【トーン】
- 共感的
- 危機感を煽りすぎない
- 希望を持たせる
```

---

## レビューチェックリスト

- [ ] タイトルで問題が明確
- [ ] ターゲットが「自分のこと」と感じる
- [ ] 問題が具体的に説明されている
- [ ] 深刻さが伝わる
- [ ] 次の記事への導線がある
- [ ] 絶望で終わらない
