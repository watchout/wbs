# 開発ログテンプレート

> 日次/週次の開発進捗を共有するフォーマット

---

## 日次開発ログ（X投稿）

### パターン1: シンプル進捗

```
Day {{N}} of building {{プロダクト名}}

今日やったこと:
- {{完了タスク1}}
- {{完了タスク2}}

明日は{{予定}}に取り組みます。

#buildinpublic #{{関連ハッシュタグ}}
```

### パターン2: 学び共有

```
{{プロダクト名}} Day {{N}}

今日の学び:
{{学んだこと}}

{{詳細な説明や気づき}}

#buildinpublic
```

### パターン3: スクショ付き

```
{{プロダクト名}} - Progress Update

{{スクリーンショットの説明}}

{{何を実装したか}}

次は{{予定}}

#buildinpublic {{画像}}
```

### パターン4: 数字共有

```
{{プロダクト名}} Week {{N}} Stats:

- ユーザー: {{数}}
- {{指標2}}: {{数}}
- {{指標3}}: {{数}}

先週比 {{+/-}}%

#buildinpublic #indiehackers
```

---

## レビューチェックリスト

- [ ] 具体的な進捗が含まれている
- [ ] 自慢しすぎていない
- [ ] 読者にとっての価値がある
- [ ] ハッシュタグが適切
- [ ] 画像/スクショがあると尚良い
- [ ] 次のアクションが明確

---

## ベストプラクティス

### DO（やるべき）

- 一貫した投稿頻度を保つ
- 失敗も正直に共有
- 数字を具体的に
- 読者の質問に返答する
- 他のBIPersと交流する

### DON'T（避けるべき）

- 自慢話ばかり
- ネガティブすぎる
- 機密情報の漏洩
- 投稿頻度の急な変化
- フィードバックの無視
