# 失敗談テンプレート

> 失敗や困難を共有し、学びに変えるフォーマット

---

## 失敗共有の価値

1. **信頼構築**: 正直さが信頼を生む
2. **共感獲得**: 同じ経験をした人との繋がり
3. **学びの共有**: 他の人の役に立つ
4. **自己整理**: 振り返りによる成長

---

### 構成

```markdown

## 何が起きたか

{{失敗の詳細な説明}}

### 背景

{{失敗に至るまでの状況}}

### 発覚

{{どうやって失敗に気づいたか}}

## なぜ起きたか

{{原因分析}}

### 直接的な原因

{{直接の原因}}

### 根本的な原因

{{なぜそうなったか、より深い原因}}

## どう対処したか

{{対処の詳細}}

### 短期的な対処

{{すぐにやったこと}}

### 長期的な対処

{{再発防止のためにやったこと}}

## 影響

{{この失敗による影響}}

- ユーザーへの影響: {{}}
- ビジネスへの影響: {{}}
- 自分自身への影響: {{}}

## 学び

{{この経験から学んだこと}}

1. {{学び1}}
2. {{学び2}}
3. {{学び3}}

## 今後の改善

{{具体的な改善策}}

## 読者へのメッセージ

{{同じ失敗をしないためのアドバイス}}
```

## AI生成プロンプト

```
以下の情報を元に、失敗談記事を生成してください。

【失敗の概要】
{{何が起きたか}}

【原因】
{{なぜ起きたか}}

【対処】
{{どう対処したか}}

【制約】
- 自己卑下しすぎない
- 学びを前向きに
- 他者批判を避ける

【出力形式】
1. X投稿用（スレッド5ツイート）
2. note記事用（2000字程度）
```

---

## レビューチェックリスト

- [ ] 事実が正確に記載されている
- [ ] 原因分析が客観的
- [ ] 自己卑下しすぎていない
- [ ] 他者を責めていない
- [ ] 学びが明確
- [ ] 読者にとって価値がある
- [ ] 機密情報が含まれていない

---

## 共有してよい失敗/避けるべき失敗

### 共有してよい

- 技術的なミス
- 判断ミス
- 計画の失敗
- コミュニケーションの失敗

### 避けるべき

- 他者のプライバシーに関わる
- 法的な問題
- 係争中の事案
- 顧客情報の漏洩につながる
