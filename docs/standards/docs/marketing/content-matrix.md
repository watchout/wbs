# コンテンツマトリクス

> マーケティング手法×チャネルの組み合わせを定義し、コンテンツ戦略を最適化する

---

## マトリクス概要

手法とチャネルの掛け合わせで、どのコンテンツをどこに配信するかを定義する。
空欄セル = その組み合わせは使用しない。

## プロジェクトタイプ別の推奨組み合わせ

### app（フルスタックWebアプリ）

```
推奨手法: 全手法対象
優先順位:
  1. BIP（開発初期〜中期）
  2. PLF（ローンチ時）
  3. DRM + 2ステップ（グロース期）
  4. ニューロ + PASONA（認知拡大）
```

### lp（ランディングページ）

```
推奨手法: DRM + PASONA + 2ステップ
理由: 直接CVを狙う構成に最適化
優先チャネル: note, X, メール
```

### hp（ホームページ）

```
推奨手法: BIP + SEO記事
理由: 長期的な信頼構築
優先チャネル: note, YouTube
```

### api / cli（開発者向け）

```
推奨手法: BIP + 技術記事
理由: 開発者コミュニティへのリーチ
優先チャネル: X, note, YouTube（技術解説）
追加: Zenn, Qiita への展開も検討
```

## 使い方

### 1. プロジェクトタイプを確認

```
.framework/profile.json の type を確認
→ 推奨組み合わせを特定
```

### 2. 手法を選択

```
プロダクトのフェーズと目的に応じて選択:
- 認知拡大 → ニューロ, BIP
- リード獲得 → 2ステップ, DRM
- ローンチ → PLF
- CV最大化 → PASONA, DRM
```

### 3. チャネルを決定

```
ターゲットの行動パターンに応じて選択:
- 情報収集型 → note, YouTube
- SNS常駐型 → X, Instagram
- メール確認型 → メールシーケンス
```

### 4. コンテンツ生成

```
strategies/{手法}/templates/ のテンプレートを使用
channels/{チャネル}.md の制約を確認
```
