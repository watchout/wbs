## SEO/AiEO対策

### SEO対策

```
1. タイトルにキーワード
2. 見出し（H2, H3）にキーワード
3. 最初の100文字に要点
4. 内部リンク（自分の記事へ）
5. 適切なタグ設定
```

### AiEO対策（AI最適化）

```
AIによる回答ソースになるために:

1. 質問形式のセクション
   「〇〇とは？」「なぜ〇〇なのか？」

2. 構造化された情報
   箇条書き、表、ステップ

3. 明確な定義
   「〇〇とは、△△のことです」

4. 網羅性
   1つのトピックを深く、広く

5. 最新情報
   更新日を記載
```

## 記事構成テンプレート

### 基本構成（2000-3000字）

```markdown
# {{タイトル}}

{{リード文: 記事で得られることを3行で}}

## 目次（自動生成されるが、手動で入れても良い）

## {{セクション1}}

{{本文}}

### {{サブセクション}}

{{詳細}}

## {{セクション2}}

{{本文}}

## {{セクション3}}

{{本文}}

## まとめ

{{重要ポイントの振り返り}}

---

この記事が参考になったら、スキ❤️をお願いします。

{{関連記事へのリンク}}
```

### フック（冒頭）のパターン

```
1. 問題提起: 「〇〇で困っていませんか？」
2. 共感: 「以前の私も〇〇でした」
3. 結論先出し: 「結論から言うと〇〇です」
4. 数字: 「〇〇を実践して△△を達成しました」
5. 物語: 「ある日、〇〇が起きました」
```

## タグ戦略

### タグの使い方

```
- 最大10個まで
- 関連性の高いもの
- 人気タグ + ニッチタグ
```

### おすすめタグ

```
ビジネス系:
#スタートアップ #起業 #ビジネス #仕事

マーケティング系:
#マーケティング #SNS #集客 #note初心者

開発系:
#プログラミング #エンジニア #開発 #テック

Build in Public:
#作ってみた #開発日記 #個人開発
```

## マガジン活用

### マガジンの使い方

```
1. テーマ別に記事をまとめる
2. 有料マガジンで収益化
3. 定期購読で継続収入
```

### マガジン設計例

```
- 開発日記（無料、Build in Public）
- 〇〇完全ガイド（有料、まとめ記事）
- 月刊〇〇レポート（定期購読）
```
