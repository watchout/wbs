# メールチャネルガイド

> メールマーケティングのベストプラクティス

---

## メール仕様

### 構成要素

| 要素 | 説明 |
|------|------|
| 件名 | 開封を促すタイトル |
| プレヘッダー | 件名の補足（受信トレイで見える） |
| 差出人名 | 信頼性の構築 |
| 本文 | メインコンテンツ |
| CTA | 行動を促すボタン/リンク |
| フッター | 配信解除リンク（必須） |

### 推奨仕様

| 要素 | 推奨 |
|------|------|
| 件名 | 30-50文字 |
| プレヘッダー | 50-100文字 |
| 本文幅 | 600px |
| CTA | 1メール1-2個 |

---

## 件名のベストプラクティス

### 効果的な件名パターン

```
1. 数字: 「3つの方法で〇〇を改善」
2. 疑問形: 「〇〇していませんか？」
3. 緊急性: 「【本日まで】〇〇」
4. パーソナル: 「{{名前}}さん、〇〇」
5. 好奇心: 「なぜ〇〇なのか」
6. ベネフィット: 「〇〇を達成する方法」
```

### 避けるべき

```
- ALL CAPS（全て大文字）
- 過度な記号（！！！、★★★）
- スパムワード（無料、今すぐ、限定）
- 長すぎる件名
- 嘘や誇張
```

### A/Bテスト案

```
- 絵文字あり vs なし
- 疑問形 vs 断言
- 短い vs 長い
- パーソナライズ vs 一般
```

## プレヘッダー

### 役割

```
- 件名を補完
- 開封率を上げる
- 受信トレイで差別化
```

### テンプレート

```
件名: 〇〇の方法をお伝えします
プレヘッダー: この方法で△△が改善しました

件名: 【残り24時間】特別オファー
プレヘッダー: 明日で終了します。お見逃しなく。
```

---

## 本文構成

### 基本構成

```
1. 挨拶（パーソナライズ）
2. フック（興味を引く1文）
3. 本文（価値提供）
4. CTA（明確な行動指示）
5. PS（追伸、補足）
```

### テンプレート

```
{{名前}}さん、

{{フック: 1-2文で興味を引く}}

{{本文: 3-5段落で価値提供}}

[{{CTA}}]({{URL}})

{{署名}}

PS: {{追伸: ボーナス情報や緊急性}}
```

## CTAパターン

### 効果的なCTA

```
- 「詳しく見る」
- 「今すぐ始める」
- 「無料でダウンロード」
- 「予約する」
- 「続きを読む」
```

### CTAボタンのデザイン

```
- 目立つ色（背景と対比）
- 十分な大きさ
- 余白を確保
- 1メール1-2個まで
```

```
Day 3: FAQ/不安解消
Day 4: 社会的証明
Day 5: 緊急性（残り48時間）
Day 6: 最終リマインド
Day 7: クロージング
```

## 分析指標

### 追跡すべきKPI

| 指標 | 計算方法 | 目標 |
|------|---------|------|
| 開封率 | 開封/配信 | 20-30% |
| クリック率 | クリック/配信 | 2-5% |
| CTOR | クリック/開封 | 10-15% |
| 解除率 | 解除/配信 | 0.5%以下 |
| CV率 | CV/クリック | 目標による |

### 改善アクション

```
開封率が低い → 件名を改善
クリック率が低い → 本文/CTAを改善
解除率が高い → 頻度/内容を見直し
```

## 法的要件

### 必須事項

```
1. 配信解除リンク（ワンクリック）
2. 送信者の連絡先
3. オプトイン取得
```

### 日本（特電法）

```
- 送信者の氏名/名称
- 送信者の連絡先
- オプトアウトの方法
```

### GDPR（EU向け）

```
- 明示的な同意
- データの用途説明
- 削除要請への対応
```
