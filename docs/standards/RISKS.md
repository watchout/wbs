## 1. リスク評価基準

### 1.1 発生確率

| レベル | 確率 | 説明 |
|-------|------|------|
| 高 | > 70% | ほぼ確実に発生する |
| 中 | 30-70% | 発生する可能性がある |
| 低 | < 30% | 発生する可能性は低い |

### 1.2 影響度

| レベル | 説明 | スケジュール影響 | コスト影響 |
|-------|------|----------------|-----------|
| 高 | プロジェクト成功を脅かす | > 2週間 | > 20% |
| 中 | 重大な影響がある | 1-2週間 | 10-20% |
| 低 | 軽微な影響 | < 1週間 | < 10% |

### 1.3 リスクスコア

```
リスクスコア = 発生確率 × 影響度

    │ 影響度
    │ 高  │  3  │  6  │  9  │
    │ 中  │  2  │  4  │  6  │
    │ 低  │  1  │  2  │  3  │
    └─────┴─────┴─────┴─────┘
          低    中    高   発生確率
```

| スコア | 優先度 | 対応 |
|-------|-------|------|
| 6-9 | 🔴 高 | 即座に対策実施 |
| 3-4 | 🟡 中 | 計画的に対策 |
| 1-2 | 🟢 低 | 監視 |

### 4.2 リスク発生時の対応

```
1. リスク発生を確認
   ↓
2. 影響範囲を評価
   ↓
3. コンティンジェンシープラン発動
   ↓
4. ステークホルダーへ報告
   ↓
5. 対応状況を追跡
   ↓
6. 振り返り・教訓化
```
