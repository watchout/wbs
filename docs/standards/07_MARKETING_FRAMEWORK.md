## Part 1: 戦略設計

### 1.1 卓越の戦略（Strategy of Preeminence）

> 「顧客の最善の利益のために行動する信頼されるアドバイザーになる」

**原則**:
- 顧客を「クライアント」として扱う（取引相手ではなく保護すべき存在）
- 売り込む前に価値を与える
- 顧客の成功 = 自分の成功

**フレームワークへの適用**:

| フェーズ | 適用方法 |
|---------|---------|
| LP | 売り込みの前に「教育コンテンツ」を提供 |
| β版 | 無料で価値を提供し、信頼を構築 |
| 有料化 | 顧客の成功にコミットした価格設計 |

---

### 1.2 3つの成長軸（ジェイ・エイブラハム）

```
売上 = 顧客数 × 客単価 × 購買頻度

各軸を10%改善 → 売上1.33倍
各軸を25%改善 → 売上1.95倍（約2倍）
```

**設計テンプレート**:

| 成長軸 | 現状 | 施策 | 目標 |
|-------|------|------|------|
| **顧客数を増やす** | | | +XX% |
| リード獲得 | | LP、SNS、広告 | |
| コンバージョン改善 | | コピー改善、保証 | |
| 紹介システム | | 紹介プログラム | |
| **客単価を上げる** | | | +XX% |
| アップセル | | 上位プラン提案 | |
| クロスセル | | 関連サービス | |
| 価格設定 | | 価値に見合った価格 | |
| **購買頻度を上げる** | | | +XX% |
| 継続率向上 | | オンボーディング改善 | |
| 再購入促進 | | メール、リマインド | |
| 習慣化 | | 日常フローへの組み込み | |

### 1.3 USP（Unique Selling Proposition）

**USP設計フレームワーク**:

```
[ターゲット] が
[競合/代替手段] ではなく、私たちを選ぶ理由は
[独自の価値] を提供するから。

なぜなら [信頼できる根拠] があるから。
```

**USPチェックリスト**:
- [ ] 競合と明確に違う
- [ ] 顧客にとって重要な違いである
- [ ] 1文で説明できる
- [ ] 証明できる（実績、データ、保証）

## Part 2: コピーライティング

### 2.1 PASONAの法則（神田昌典）

```
P - Problem（問題）    : 顧客の痛みを明確にする
A - Agitation（煽り）  : 問題を放置した場合の結果を描く
S - Solution（解決策） : 解決策を提示する
O - Offer（提案）      : 具体的なオファーを提示する
N - Narrow（絞り込み） : 今すぐ行動すべき理由を示す
A - Action（行動）     : 具体的な行動を促す
```

**LP構成への適用**:

| セクション | PASONA | 内容 |
|-----------|--------|------|
| ヘッドライン | P | 「○○で悩んでいませんか？」 |
| 問題提起 | A | 「このまま放置すると...」 |
| 解決策 | S | 「[製品名]なら解決できます」 |
| 機能・ベネフィット | O | 「具体的には...」 |
| 限定性 | N | 「β版は100名限定」 |
| CTA | A | 「今すぐ登録する」 |

---

### 2.2 感情トリガー（ニューロマーケティング）

**購買を促す8つの感情トリガー**:

| トリガー | 説明 | コピー例 |
|---------|------|---------|
| **恐怖** | 失うことへの恐れ | 「このままでは競合に負けます」 |
| **欲求** | 得たいという願望 | 「理想の状態を手に入れる」 |
| **希少性** | 手に入らなくなる恐れ | 「残り3枠」 |
| **緊急性** | 今すぐ行動すべき理由 | 「本日23:59まで」 |
| **社会的証明** | 他の人も使っている安心 | 「1,000社が導入」 |
| **権威** | 専門家のお墨付き | 「○○大学教授推薦」 |
| **好意** | 共感・親近感 | 「私も同じ悩みを持っていました」 |
| **一貫性** | 以前の行動との整合性 | 「無料登録した方限定」 |

---

### 2.3 脳に刺さるコピーの原則

**原則1: 爬虫類脳に訴える**

```
脳の構造:
1. 爬虫類脳（本能）  ← ここに最初に訴える
2. 哺乳類脳（感情）
3. 人間脳（理性）

爬虫類脳が反応するもの:
- 自己保存（生存、安全）
- 快楽追求（楽したい）
- 苦痛回避（面倒を避けたい）
```

**原則2: 具体性**

| ❌ 曖昧 | ✅ 具体的 |
|--------|---------|
| 売上アップ | 売上147%アップ |
| 多くの企業 | 1,247社 |
| すぐに | 3分で |

**原則3: ベネフィット > 機能**

| ❌ 機能 | ✅ ベネフィット |
|--------|---------------|
| AIによる自動分析 | 毎日2時間の作業が不要に |
| クラウド保存 | どこからでもアクセス、データ消失の心配ゼロ |

## Part 3: ファネル設計

### 3.1 2ステップマーケティング（DRM）

```
【1ステップ】いきなり売る
訪問者 → 購入 （CV率: 1-3%）

【2ステップ】まず関係構築
訪問者 → リード獲得 → 育成 → 購入 （CV率: 10-30%）
           ↓
        リードマグネット
        （無料で価値提供）
```

**リードマグネット設計**:

| タイプ | 例 | 効果 |
|-------|-----|------|
| チェックリスト | 「LP作成77項目チェックリスト」 | 高（作成容易） |
| テンプレート | 「コピペで使えるメールテンプレ10選」 | 高 |
| ミニ講座 | 「3日間メール講座」 | 中（育成も兼ねる） |
| ツール | 「ROI計算シート」 | 高（実用的） |
| レポート | 「業界動向レポート2024」 | 中 |
| ウェビナー | 「無料ウェビナー」 | 高（信頼構築） |

---

### 3.2 フロントエンド/バックエンド戦略

```
フロントエンド        バックエンド
（集客商品）          （収益商品）
    ↓                    ↓
低価格/無料     →    高価格/継続課金
価値提供              本命商品
信頼構築              利益確保
```

**設計テンプレート**:

| 段階 | 商品 | 価格 | 目的 |
|------|------|------|------|
| 無料 | リードマグネット | ¥0 | リード獲得 |
| フロントエンド | お試しプラン/低価格商品 | ¥XXX | 顧客化、信頼構築 |
| ミドル | 通常プラン | ¥XXX/月 | 収益の柱 |
| バックエンド | 上位プラン/コンサル | ¥XXX | 利益最大化 |

---

## Part 4: ローンチ戦略

### 4.1 プロダクトローンチ・フォーミュラ

```
Pre-Pre Launch → Pre Launch → Launch → Post Launch
    種まき        期待醸成     販売      フォロー

【期間目安】
2週間前        1週間前       3-7日      1週間
```

**各フェーズの施策**:

| フェーズ | 施策 | 内容 |
|---------|------|------|
| **Pre-Pre Launch** | ティーザー | 「何か新しいことを始めます」 |
| | アンケート | 「どんな機能が欲しいですか？」 |
| | カウントダウン | 「あと14日」 |
| **Pre Launch** | コンテンツ提供 | 無料で価値を提供（動画、記事） |
| | ストーリー | 開発の裏側を共有 |
| | 社会的証明 | β版ユーザーの声 |
| **Launch** | オープン告知 | 「ついに公開！」 |
| | 限定オファー | 早期割引、特典 |
| | 緊急性 | 「72時間限定」 |
| **Post Launch** | サポート | オンボーディング |
| | 証言収集 | 成功事例の収集 |
| | 次への布石 | アップセル準備 |

---

### 4.2 ライブローンチ（小山竜央式）

**ライブイベントの力**:
- リアルタイムの熱量
- 双方向コミュニケーション
- 限定感・緊急性
- コミュニティ感

**ローンチイベント設計**:

| 要素 | 内容 |
|------|------|
| 形式 | ウェビナー / YouTube Live / Twitter Space |
| 時間 | 60-90分 |
| 構成 | 価値提供(70%) + オファー(30%) |
| 特典 | ライブ参加者限定特典 |

**ウェビナー構成**:

| 時間 | 内容 | 目的 |
|------|------|------|
| 0-5分 | 自己紹介、今日の流れ | 関係構築 |
| 5-15分 | 問題提起、共感 | Pain の確認 |
| 15-40分 | 解決策、ノウハウ提供 | 価値提供、信頼構築 |
| 40-50分 | 成功事例、証言 | 社会的証明 |
| 50-60分 | オファー、Q&A | 行動促進 |

## Part 5: 紹介システム

### 5.1 紹介プログラム設計（ジェイ・エイブラハム）

> 「満足した顧客は最高のセールスパーソン」

**紹介が起きる条件**:
1. 製品/サービスに満足している
2. 紹介するインセンティブがある
3. 紹介しやすい仕組みがある

**紹介プログラムテンプレート**:

| 項目 | 内容 |
|------|------|
| 紹介者への特典 | 例: 1ヶ月無料、クレジット付与 |
| 被紹介者への特典 | 例: 初月50%OFF |
| 紹介方法 | 専用リンク、紹介コード |
| 追跡方法 | |
| 支払いタイミング | 登録時 / 課金時 |

---

### 5.2 口コミを生む仕組み

**口コミが発生する瞬間（WOWモーメント）**:

| タイミング | 施策 |
|-----------|------|
| 初回利用時 | オンボーディングで「すごい」体験 |
| 成果が出た時 | 成果の可視化、共有機能 |
| サポート対応 | 期待を超えるサポート |
| アップデート時 | 要望した機能の実装 |

**シェアされやすくする仕組み**:
- ワンクリックでTwitter投稿
- 成果のシェア画像自動生成
- ハッシュタグの統一
